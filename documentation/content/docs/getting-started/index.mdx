Use this page when you want the minimum end-to-end flow:

- open the production dashboard;
- sign in with a wallet;
- add a protocol to your workspace;
- read the score in the dashboard;
- create a token for the SDK.

## 1. Open the dashboard

Use:

```txt
https://stps-client.vercel.app/
```

Then sign in with the wallet you want to use for this STPS workspace.

## 2. Add the protocol

In the dashboard, go to `Protocols` and provide:

- `Program address`: the Solana program ID of the protocol;
- `Workspace label`: an optional display name.

This creates an account-scoped watchlist entry. STPS does not require you to
prove that the wallet owns or controls the protocol before adding it to your
workspace.

## 3. Create an SDK token

In the dashboard token area, create a persistent token.

The raw value is shown only once. Store it as a secret in your backend or CI:

```bash
STPS_API_TOKEN=stps_...
```

## 4. Read protocols through the SDK

```ts
import { StpsClient } from "stps-sdk";

const client = new StpsClient({
  token: process.env.STPS_API_TOKEN!,
});

const { protocols } = await client.getProtocols();

for (const item of protocols) {
  console.log(item.protocolAddress, item.protocol.currentScore, item.status);
}
```
