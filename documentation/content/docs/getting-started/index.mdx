Use this page when you want the minimum end-to-end flow:

- open the production dashboard;
- sign in with a wallet;
- claim a protocol;
- verify control;
- read the score in the dashboard;
- create a token for the SDK.

## 1. Open the dashboard

Use:

```txt
https://stps-client.vercel.app/
```

Then sign in with the wallet linked to the protocol.

> If you use a different wallet, the claim may still be created, but control
> verification may fall back to `manual_review`.

## 2. Claim the protocol

In the dashboard, go to `Protocols` and provide:

- `Program address`: the Solana program ID of the protocol;
- `Workspace label`: an optional display name.

The claim creates the link between the operator account and that protocol.

## 3. Verify control

The strong verification path currently accepts:

- the signing wallet matches the program upgrade authority; or
- the signing wallet belongs to the signer/admin set known by STPS.

Possible states:

| Status | Meaning |
| --- | --- |
| `claimed` | Protocol linked to the account, but control is not yet proven. |
| `verified` | Control proven through a supported method. |
| `manual_review` | STPS could not prove control automatically. |

## 4. Create an SDK token

In the dashboard token area, create a persistent token.

The raw value is shown only once. Store it as a secret in your backend or CI:

```bash
STPS_API_TOKEN=stps_...
```

## 5. Read protocols through the SDK

```ts
import { StpsClient } from "stps-sdk";

const client = new StpsClient({
  token: process.env.STPS_API_TOKEN!,
});

const { protocols } = await client.getProtocols();

for (const item of protocols) {
  console.log(item.protocolAddress, item.protocol.currentScore, item.status);
}
```

## Surfaces used in this flow

- Dashboard:
  `https://stps-client.vercel.app/`
- SDK:
  package `stps-sdk`
