## Public surfaces

Today STPS is publicly consumed through two surfaces:

- Dashboard: `https://stps-client.vercel.app/`
- SDK: `https://www.npmjs.com/package/stps-sdk`

## How to consume the product

- Human operator:
  use `https://stps-client.vercel.app/`
- Backend integration:
  use `stps-sdk` with a persistent token

## Self-hosted environment

If a team wants to run its own Scoring Engine, it can:

- run the STPS backend with its own database;
- issue its own persistent tokens;
- point the SDK to its own `apiUrl`;
- optionally point a custom frontend to the same engine.

The dashboard published at `stps-client.vercel.app` will continue to show the
engine configured for that app, not the engine a user chooses independently in
their SDK integration.

## Surface choice

In practice, there are two adoption paths:

- use the official dashboard for human operations;
- use the SDK for integration into internal services, bots, or monitors.

The rest of the architecture exists to support those two surfaces.
