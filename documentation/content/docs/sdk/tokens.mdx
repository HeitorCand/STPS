SDK tokens are persistent credentials tied to an STPS account.

They are not wallet session tokens.

## Properties

- The token does not expire automatically.
- The token can be revoked in the dashboard.
- The raw token is shown only once, at creation time.
- The database stores only the token hash.
- Each token can see only the protocols linked to the account.

## Header used by the SDK

```txt
X-STPS-Token: <token>
```

## Best practices

- Use it in backend systems, CI, or internal services.
- Do not expose it in a public frontend.
- Name the token with a clear label, such as `backend-prod` or `ci-monitor`.
- Revoke tokens that are no longer used.

## Custom engine

The token is only valid in the Scoring Engine that issued it.

If you use a custom `apiUrl` in the SDK, that engine must have:

- the same Supabase schema;
- the same workspace-facing contract;
- tokens issued in that environment.
