Use the SDK when a backend, bot, monitor, or internal service needs to read the
protocols linked to an STPS account.

The SDK is not an embedded scoring engine. It is a client for consuming trust
state that STPS has already decided.

## Installation

```bash
npm install stps-sdk
```

The published package is available on npm as `stps-sdk`.

Package page:

```txt
https://www.npmjs.com/package/stps-sdk
```

## Token

Create a token in:

```txt
/dashboard/tokens
```

Store it as a secret:

```bash
STPS_API_TOKEN=stps_...
```

That token represents the operator account in the SDK context. Because of that,
the SDK only sees the protocols linked to that account, not a global public
catalog.

## First script

```ts
import { StpsClient } from "stps-sdk";

const client = new StpsClient({
  token: process.env.STPS_API_TOKEN!,
});

const { protocols } = await client.getProtocols();

for (const item of protocols) {
  console.log({
    protocol: item.protocolAddress,
    status: item.status,
    score: item.protocol.currentScore,
    risk: item.protocol.riskLevel,
  });
}
```

## Using the official product

If you do not pass `apiUrl`, the SDK automatically uses the official STPS
environment. For standard product consumption, that is the expected path.

## Using your own Scoring Engine

If you run your own engine:

```ts
const client = new StpsClient({
  token: process.env.STPS_API_TOKEN!,
  apiUrl: "https://my-scoring-engine.example.com",
});
```

In that scenario, your engine needs to reproduce the same conceptual product
logic:

- maintain a certificate model per protocol;
- expose an account/workspace layer;
- accept persistent tokens tied to the account;
- return only the protocols allowed for that token;
- remain compatible with the SDK read contract.

In other words, pointing `apiUrl` elsewhere is not just swapping hosts. You are
stating that another STPS-compatible system exists on the other end.

The frontend will only show a custom engine if it is also configured to use
that URL. The SDK does not automatically change the dashboard URL.

## Health check

```ts
const ok = await client.isHealthy();
```

This method validates that the API is available and the token is accepted.
